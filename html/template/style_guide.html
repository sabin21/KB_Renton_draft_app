ID, Class 네이밍 기본 규칙

  • ID는 카멜 케이스(ex: headerWrap) 규칙을 기본으로 사용한다.
  • Class는 하이픈(-)을 사용한다. (ex: btn-circle)
  • 영문, 숫자만 사용할 수 있다.
  • 활성활 상태를 나타낼 경우 Class이름을 active로 사용한다.

이미지 네이밍

  • 이미지 예약어는 네이밍 규칙에 근거하여 작성한다.
  • 같은 분류의 이미지가 두 개 이상이면 파일 이름 마지막에 숫자를 추가하여 구분한다.
  • 이미지 네이밍을 확장해야 할 때는 이미지 네이밍 조합 규칙을 준수한다.
  • 의미를 잘 살린 이미지의 naming은 이미지를 보지 않아도 대강의 의미를 알 수 있다.
  • 이는 규모가 큰 프로젝트일 경우 다수의 작업자들이 중복된 이미지를 저장하는 것에 대한 예방을 목적으로 한다.

html

  • 전체적인 마크업은 HTML5 을 기본으로 한다.
  • inline 스타일(< div style="....." > )은 되도록이면 자제

    <meta http-equiv="Content-type" content="text/html; charset=utf-8" />

  • 의미에 맞는 구조 설계 (Semantic Markup)

    Table 레이아웃을 배제하며 의미있는 구조의 태그를 사용합니다.
    <h1>(머릿말), <p>(문단), <ul>(목록), <dl>(용어 정의 목록)

  • 종료 태그 표기

    HTML은 종료태그가 없는 것을 허용하였으나 XHTML은 반드시 종료태그를 갖습니다.
    HTML은 태그의 중첩이 잘못된 것을 허용하였으나 XHTML은 잘못된 중첩을 허용하지 않습니다.
    </p> <input /> <img />

  • 요소와 속성은 소문자로 표기

    HTML은 요소와 속성에 대소문자를 함께 사용하는 것을 허용하였으나 XHTML의 마크업 '요소'와 '속성'들은 반드시 소문자로 표기되어야 합니다. 단, 속성의 '값'에는 대소문자 혼합 표기가 가능합니다. 하지만 대소문자를 명확하게 구분하기 때문에 대문자로 구성된 '값'과 소문자로 구성된 '값'은 동일하지 않고 확실히 구별됩니다.

  • 속성값은 따옴표로 감싸서 표기

    HTML의 경우 속성 값들을 따옴표로 감싸지 않는 것을 허용하였지만 XHTML 에서는 반드시 속성 "값"은 따옴표 안에 있어야 합니다.

  • 특수문자는 Entity 코드로 변환

    " (&quot;), &(&amp;), <(&lt;), >(&gt;)

  • 모든 이미지와 이미지 맵에는 alt, title 속성을 부여해 주어 해당 이미지와 컨텐츠의 의미를 반영

    <img class="pic" src="../../../../images/mokpo/kr/common/ic_new.gif" alt="new" />

  • 0을 제외한 모든 값들은 단위를 표기

    width="0" height="150px"

  • Html 주석 가이드

    그룹화하는 시작과 끝에 삽입하며 주석과 관련된 버그를 피하기 위하여 주석 기호와 코멘트 사이에는 빈 공간을 두어야 합니다.
    <!-- s:주석시작 -->
    <!-- //e:주석끝 -->

button

접근성 주의사항

  • URL를 ‘연결‘하기 위한 버튼으로서 <a href="#url"> 형식으로 마크업하며 a 태그에는 반드시 href 속성을 넣어준다.(부득이하게 onclick 등으로 작동될 경우 웹접근성을 위해 href="#의미" 로 작성)
  • href 속성이 없거나 href="" 일 경우 스크린리더가 버튼으로 인식 못함.
  • 새창이 열리는 버튼일 경우 웹접근성 작업으로 title="새창열림" 을 반드시 넣어준다.
  • URL를 연결하지 않고 단순하게 사용자 인터페이스만를 ‘조작‘하기 위한 버튼으로서 <button type="button"> 엘리먼트로 마크업 합니다. type을 명시하지 않으면 어떤 브라우저는 Form Submit 기능을 실행하기 때문에 반드시 type 속성을 button으로 지정해 주어야 합니다.

접근성 주의사항

  • label은 눈에 보이지 않게 제공할 수 있고, 폼 콘트롤 요소에 title로도 제공 가능함. label의 for와 input의 id 일치

  • 매칭할 대체 텍스트가 없는 경우 input에 title 제공
