Naming 규칙
공통 규칙
naming은 영문 소문자, 숫자, 하이픈( - )으로 작성한다.
naming의 첫 시작은 영문 소문자로만 시작한다.
id는 카멜표기법, class는 하이픈( - )방식을 사용한다.
예) tabTy1, tab-ty1
단어조합시, [형태+의미+숫자+상태]의 형태로 작성한다.
숫자가 없으면 '1'이라는 숫자가 생략된 것으로 간주한다.
동일한 이름의 id는 문서에서 한번만 사용이 가능하다.
다중의 클래스 지정을 허용하며 최대 4개 이상 사용하지 않도록 주의한다.
약속어를 기본으로 작성하나, 없으면 종류와 특성을 나타내도록 naming한다.
naming 확장
naming 확장시 기존의 class 에 타입을 추가한다.
첫번째 파일은 숫자를 생략하여 사용할 수 있다.
예) box, box ty2, box ty3
약속어
객체, 이미지, 파일 및 네이밍 시 의미를 일관되게 표현하기 위해 정의
타이틀 tit- 작은타이틀 tit-dep1, tit-dep2
텍스트 txt- 탭 tab-
버튼 btn- 이미지 img-
배경 bg- 스프라이트 이미지 sp-
아이콘 ico- 팝업 pop-
상태변화 -off, -on, -over 임시파일 tmp-
파일 naming 조합
HTML파일은 화면ID로 작업한다.
예) 화면ID.html
파일의 인코딩
*.html, *.css, *.js의 파일의 저장 방식을 UTF-8로 설정하되 개발팀과 협의하여 결정한다.
코드 들여쓰기
코드의 가독성을 높이고 전체 HTML 구조를 쉽게 파악하기 위하여 들여쓰기 규칙을 준수한다. 마크업의 중첩이 깊어질 때마다 자식 엘리먼트는 1탭 들여 쓰고, 1탭의 크기는 공백 4칸으로 설정한다.

웹표준/웹접근성 적용
개발시 유의할점
input, select, textarea의 경우 label과 연결
<label for="name">이름</label><input type="text" id="name">
적절한 label을 연결할 수 없을 경우 title=""제공 : <input type="text" title="검색어입력">
<img>일 경우 : alt="이미지와 동일한 내용입력" 제공.
새창이 떠야할 경우 새창임을 알림 : target="_blank" 일경우 title="새창" 제공.
